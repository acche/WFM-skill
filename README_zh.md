## 🌟 简介

本项目是一个专为呼叫中心 WFM（排班与劳动力管理）及 RTA（实时监控）专员设计的 AI 辅助系统。覆盖呼叫中心运营管理的完整闭环：话务预测、排班策略、实时调度、数据报表、考勤管理、招聘预警。

**核心原则：系统无关，数据驱动。**
- 有对接系统（如七陌、Genesys、NICE等）时，直接读取 API 数据流。
- 无系统对接时，支持上传 Excel/CSV 等文件，AI 将自动完成所有数据解析、计算和决策建议。
- 任何情况下，计算和规则匹配由 AI 完成，人类专员只需做**判断和审批**。

## ✨ 核心能力

- 📈 **话务量预测与分析**：结合历史数据、业务动态与外部因素，建立动态预测模型。
- 📅 **排班策略制定**：统筹客服资源，实现半小时颗粒度的弹性排班。
- ⚡ **日常排班管理与现场运营（RTA）**：7×24 全渠道闭环监控 SL/AHT/排队时长/坐席状态。
- 🚨 **人效管理与异常预警**：监控利用率、接起率，触发人力招聘及排班预警，应对突发事件。
- 📊 **数据驱动运营**：自动生成各类数据看板并扫描异常，辅助持续改善。

## 🚀 使用模式

### 模式 A：有系统 API（自动模式）
系统持续推送数据，AI 实时计算指标并监控。当数据超过安全阈值时，自动向专员推送告警及应对方案。

### 模式 B：手动上传 Excel（半自动模式）
用户上传报表文件（支持 `.xlsx` / `.csv` / 甚至截图），AI 立即执行：
1. 自动识别列结构与字段含义
2. 自动计算所有可计算的运营指标 (SL, AHT, DF% 等)
3. 标出异常项并高亮风险
4. 给出排班调配或应对建议

## 🧠 AI 能力地图

| 场景 | 需要上传的数据 | AI 处理内容 |
|------|-------------|------------|
| **实时监控** | 坐席状态表 + 时段话务量表 | 持续计算 SL/AHT/利用率，超阈值自动推送方案 |
| **话务预测** | 历史话量表（至少4周） | 识别周/月规律，叠加业务事件，输出预测量 |
| **排班优化** | 当前排班表 + 话务预测量 + 名单 | 推荐高性价比、服务优先等多种排班方案 |
| **午餐/休息调度** | 当前在线坐席数 + 话量 + 目标SL | 计算安全离岗人数，生成最优分批建议 |
| **异常数据报表** | 任意原始通话记录表 | 自动扫描缺失数据、严重超载、闲置等异常 |
| **恶劣天气/突发缺勤** | 员工档案 + 缺勤名单 | 按距离和资质自动分类，生成补班/居家名单 |

## 🤝 人机协作边界

1. **AI 负责**：海量数据计算、多维度监控、报表异常识别、生成行动草稿和预测模型。
2. **人类负责**：综合判断、一键审批、跨部门协调沟通与员工关系维护。
